# Go Board Memory Research

AI 코딩 에이전트가 장기 프로젝트에서 컨텍스트를 잃거나, 이미 완료한 작업을 반복하거나, 전체 구조를 다시 이해하려고 하면서 컨텍스트가 비대해지는 문제를 줄이기 위한 연구 노트입니다.

이 저장소는 단순한 자동매매 봇 구현 저장소가 아니라, 장기 AI 개발 작업을 안정적으로 관리하기 위한 **컨텍스트 운영 방식**을 연구하는 공간입니다.

## 핵심 아이디어

**Go Board Memory**는 바둑판의 개념을 AI 프로젝트 관리에 접목하는 방식입니다.

- 돌 = 작업 노드
- 집 = 안정화된 모듈 영역
- 자유도 = 남은 의존성 또는 미해결 조건
- 단수 = 깨지기 쉬운 위험 상태
- 패 = 반복되면 안 되는 설계 논쟁 또는 금지 결정
- 다음 수 = 현재 가장 가치 있는 단일 작업

목표는 AI에게 전체 프로젝트를 매번 읽히는 것이 아니라, 현재 좌표와 주변 좌표만 읽게 하여 컨텍스트 사용량을 제한하는 것입니다.
